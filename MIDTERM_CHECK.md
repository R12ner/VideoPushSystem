# 视频推送系统中期检查总结

## 一、项目概述

本课题围绕“视频内容分发与个性化推荐”展开，设计并实现了一个前后端分离的视频推送系统。系统面向普通用户、视频创作者和管理员三类角色，核心目标是完成从视频上传、审核、发布、播放、互动，到推荐分发、创作者管理、后台运营的一体化业务流程。  

目前系统已经完成整体技术架构搭建，形成了较完整的业务闭环。前端采用 Vue 3 + Vite + Pinia + Element Plus 实现交互界面，后端采用 Flask + SQLAlchemy + MySQL 实现接口与数据管理，同时集成了 Two-Tower 召回模型、通知机制和支付认证流程，具备较强的工程实现基础。

## 二、已完成的主要内容

### 1. 完成系统总体架构设计与开发环境搭建

- 完成前后端分离架构设计，明确了用户端、创作者端、管理端三类业务入口。
- 完成数据库表结构设计，建立了用户、视频、行为日志、评论、关注、播放列表、订单、通知、密码重置申请等核心数据模型。
- 完成 Flask 后端服务、Vue 前端工程、MySQL 数据库以及 Docker Compose 部署方案的搭建。
- 完成接口文档集成，系统已支持 Swagger/Redoc 形式的接口说明与调试。

### 2. 完成用户基础功能模块

- 实现用户注册、登录、邮箱验证码校验等身份认证功能。
- 实现个人资料管理，包括用户名、头像、横幅图、简介和密码修改。
- 实现忘记密码与管理员审核式重置密码流程，增强账户找回能力。

### 3. 完成视频内容管理模块

- 实现视频上传功能，支持普通上传和分片上传两种方式。
- 实现视频分片合并、服务端文件保存和基础元数据写入。
- 实现自动抽帧生成缩略图，减少手工处理成本。
- 实现视频时长识别，并依据视频时长和画面比例自动识别 Shorts 短视频。
- 实现视频详情编辑，包括标题、简介、分类、标签、封面、字幕、公开范围等内容维护。
- 实现草稿保存、提交发布、撤回草稿、删除视频等创作者内容管理流程。
- 实现字幕上传与片尾推荐视频配置等增强型视频发布功能。

### 4. 完成视频播放与交互模块

- 实现视频详情页播放功能，支持正常播放、暂停、跳转和进度跟踪。
- 实现观看历史记录与播放进度保存，用户再次进入视频时可以恢复上次观看位置。

### 5. 完成个性化推荐与内容分发模块

- 实现首页推荐接口，支持登录用户与未登录用户两种推荐策略。
- 已接入 Two-Tower 召回模型，能够根据用户近期行为日志生成候选视频集合。
- 实现基于行为日志的推荐输入构建，包括观看、点赞、收藏等行为记录。
- 实现热门内容兜底机制，当个性化召回不足时自动补充热门视频，提高推荐稳定性。
- 实现首页视频分类动态排序，能够依据用户近期偏好调整分类展示顺序。
- 完成短视频内容与普通视频内容的区分展示。

### 6. 完成创作者工作台模块

### 7. 完成后台管理模块

- 实现视频审核、视频删除、用户列表查询、用户封禁/解封等后台管理功能。
- 实现密码重置申请列表和管理员发送重置邮件流程。

### 9. 完成通知机制与系统联动

## 三、系统目前已经实现了什么

从整体运行流程上看，当前系统已经可以完成以下完整业务链路：

1. 新用户可以通过邮箱验证码完成注册，并通过登录进入系统。
2. 创作者可以在前端上传视频，系统会自动保存文件、生成缩略图、识别时长，并建立视频记录。
3. 创作者可在发布页面补充标题、简介、分类、标签、封面、字幕、片尾推荐视频等信息，并将视频保存为草稿或提交审核。
4. 管理员可在后台查看待审核视频，对视频进行审核通过、下架或删除处理。
5. 审核通过后，公开视频可以进入首页列表、搜索结果、频道页和推荐页中展示。
6. 普通用户可在详情页观看视频，进行点赞、收藏、评论、回复、关注作者等互动操作。
7. 系统会记录用户观看、点赞、收藏等行为，并用于后续推荐计算与观看历史展示。
8. 推荐模块会结合用户近期行为与召回模型，在首页生成个性化推荐内容；当个性化结果不足时，再由热门内容进行补充。
9. 创作者可以在工作台查看自己频道的内容数据和运营数据，并管理视频作品。
10. 管理员可以从后台查看平台整体运行情况，处理用户与内容审核事务。

也就是说，系统当前已经不只是静态页面演示，而是实现了“上传 - 审核 - 发布 - 播放 - 互动 - 推荐 - 管理”的核心闭环，具备了毕业设计中期检查所要求的阶段性成果。

## 四、系统功能详细说明

### 1. 用户端功能

用户端主要服务于普通观看者，已实现以下功能：

- 首页视频流展示：系统可展示推荐视频、短视频内容和分类内容，支持搜索与分类切换。
- 视频播放页：支持视频播放、封面展示、字幕加载、片尾推荐、相关推荐展示。
- 互动行为：支持点赞、收藏、评论、评论回复、评论点赞、关注创作者等。
- 历史记录：系统能够记录观看进度，并在再次进入视频时恢复播放位置。
- 个人中心：用户可查看和修改个人资料，查看收藏记录、观看历史等。
- 频道访问：用户可进入创作者频道，查看频道信息、公开视频和播放列表。

### 2. 创作者端功能

创作者端主要面向上传者和内容生产者，已实现以下功能：

- 视频上传：支持大文件分片上传，提高上传稳定性。
- 自动处理：系统上传后能够自动抽取缩略图、识别时长，并判断是否为短视频。
- 视频编辑：创作者可完善标题、简介、标签、分类、字幕、公开范围等信息。
- 草稿与发布：支持先保存草稿，再提交审核，符合真实视频平台内容发布流程。
- 内容管理：创作者可在工作台查看全部作品，了解当前作品状态并进行编辑、删除、重新提交等操作。
- 数据统计：工作台可以展示作品数量、播放量、粉丝量、点赞量、观看时长等指标。
- 通知设置：创作者可设置是否接收订阅、互动、评论、推荐和系统通知。

### 3. 管理端功能

管理端主要面向平台运营与审核人员，已实现以下功能：

- 平台数据总览：统计用户数、视频数、待审核视频数、分类分布和热门视频。
- 视频审核：管理员可对上传视频进行审核、下架和删除。
- 用户管理：可查看用户信息，并进行封禁和解封处理。
- 密码找回辅助管理：管理员可接收用户密码重置申请并发送重置邮件。

### 4. 推荐模块功能

推荐模块是本系统的核心特色之一，目前已完成以下实现：

- 使用行为日志记录用户对视频的观看、点赞、收藏等行为。
- 基于用户近期行为序列构建推荐输入。
- 加载预训练的 Two-Tower 召回模型，对候选视频进行召回。
- 将召回结果与本地视频数据结合，生成首页推荐列表。
- 在召回结果不足时通过热门内容进行兜底，保证首页始终有内容可展示。
- 根据用户近期观看分类调整首页分类顺序，使推荐界面更具个性化特征。

### 5. 支付与认证功能

系统在基础视频平台功能之外，还扩展了支付与认证能力：

- 用户可发起认证订单。
- 系统通过支付宝沙箱接口生成支付链接。
- 支付成功后由异步回调或主动查单同步订单状态。
- 系统根据支付结果更新用户认证类型，并在前端显示认证标识。

这一部分体现了系统在功能设计上并不仅停留于基础的“看视频”，而是考虑了账户等级与平台扩展场景。
